# Title link correction — vacancy and freelance Telegram reports

**Date:** 2026-06-25
**Skill:** research/vacancy-monitoring and research/freelance-marketplace-monitoring
**User correction:** job titles in Telegram reports must link to the original listing on the source platform.

## Symptom

The Upwork daily report displayed job titles as plain bold text, e.g.:

```
1. *AI Engineer / Automation Specialist: Help a Sports Fan-Engagement Startup Go AI-Native*
💰 $40.00-128.00/hr  |  🎯 Expert  |  🕐 yesterday
```

Only the master-sheet link at the top was clickable; the individual job title was not a link.

## Root cause

The report formatter used Markdown bold (`*{title}*`) and placed the URL only in a separate Google Sheets column, not in the Telegram message. The cron dispatcher does not set `parse_mode=Markdown`, so even if `[text](url)` had been used it would have rendered as raw text. The correct fix is to output HTML with an `<a>` tag around each title.

## Fix

Changed the formatter in the relevant scraper so each title is wrapped in an HTML link:

```python
# Old
lines.append(f"{i}. *{title}*\\n💰 {j['rate']} ...")

# New
safe_title = title.replace(']', ' ').replace('[', ' ').replace('\\\\', ' ')
lines.append(f"{i}. [{safe_title}]({j['url']})\\n💰 {j['rate']} ...")
```

For vacancy-monitoring reports (which already used HTML), the equivalent rule is:
- `build_report()` must emit `1. <a href=\"{h(v['url'])}\"\u003e{h(v['title'])}</a>` in the TOP section.
- By-source list must also use `<a href=\"...\"\u003e` for each title.
- Dynamic text is escaped with `html.escape()` before insertion.

## Cross-skill rule

Every Telegram report generated by `vacancy-monitoring` or `freelance-marketplace-monitoring` must make the item title a clickable link to the original board page. This applies to:
- Upwork (`freelance-marketplace-monitoring`)
- HH.ru / Superjob / Habr / LinkedIn / remote boards (`vacancy-monitoring`)
- Any future platform added to either umbrella
